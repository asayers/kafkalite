data Segment Source

Kafka streams are split into segments on disk. Each segment comprises a log file, which stores the data, and an index, which allows fast seeking to a specific message within the log file. The files associated with a segment are named according to the offset (within the whole topic) of the first message in the segment. The message offsets used in the index are given relative to the starting offset of the segment.

type Index = Vector IndexEntry Source

An unboxed vector of offset-position relations, which is

  • sorted in both columns; and
  • sparse in both columns.

data IndexEntry Source

Relates a logical message offset (given relative to the start of the segment) to a byte offset within the log file for the segment.
